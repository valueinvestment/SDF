# 제품 요구사항 문서 (PRD)
# SDF 디지털 트윈 멀티 에이전트 시뮬레이터

**버전:** 1.0  
**작성일:** 2026-05-27  
**작성자:** 최승훈  
**상태:** 개발 승인됨

---

## 1. 요약

SDF(Software Defined Factory) 디지털 트윈 멀티 에이전트 시뮬레이터는 두 가지 역량을 동시에 입증하기 위한 풀스택 포트폴리오 프로젝트입니다. 대규모 프론트엔드 렌더링 최적화(Three.js + WebSocket + ECharts)와 Claude API 멀티 에이전트 파이프라인을 활용한 프로덕션 수준의 AI 오케스트레이션이 핵심입니다. 실제 공장 바닥을 3D로 시뮬레이션하고, 기계 및 로봇의 실시간 센서 데이터를 스트리밍하며, 고장을 진단하고 수리 로봇을 파견하고 RICE 기반 실행 계획을 생성하는 자율 AI 에이전트 체인을 실시간으로 시각화합니다.

---

## 2. 문제 정의

기업 소프트웨어 엔지니어에게는 고성능 프론트엔드 시스템과 AI 통합 모두에 대한 역량 증명이 점점 더 요구되고 있습니다. 둘 중 하나만 보여주는 포트폴리오 프로젝트는 흔합니다. 이 프로젝트는 스마트 제조 / 디지털 트윈이라는 기술적으로 신뢰할 수 있는 도메인 안에서 두 가지를 하나의 일관된 제품으로 증명합니다.

---

## 3. 목표 및 성공 기준

### 핵심 목표
| 목표 | 측정 지표 |
|---|---|
| 프론트엔드 렌더링 최적화 증명 | Three.js 씬이 메모리 누수 없이 24시간 안정적인 60fps 유지 |
| AI 오케스트레이션 깊이 증명 | A→B→C 에이전트 체인이 실행되어 구조화된 출력을 UI에 표시 |
| 공개 접근 가능 | 배포된 URL이 콜드 스타트 후 5초 이내에 동작하는 데모 반환 |
| 포트폴리오로서 기술적 인상 | 에이전트 체인 출력이 구체적이고 의미 있는 내용 (형식적 텍스트 X) |

### 비목표
- 이 프로젝트는 실제 공장 제어 시스템이 아님
- 실제 하드웨어 연동 또는 OPC-UA/MQTT 연결 없음
- 사용자 인증 또는 멀티테넌트 지원 없음
- 센서 이력 또는 보고서를 위한 영구 DB 없음

## 4. 사용자 페르소나

**주요 사용자: 엔지니어링 채용 담당자 / 기술 리크루터**
- 공개 URL을 통해 프로젝트를 확인, 일반적으로 2~5분 체류
- 평가 기준: 이게 기술적으로 실제인가? 깊이가 있는가? 풀스택 시스템을 구축할 수 있는가?
- 필요: 첫 로딩부터 시각적으로 인상적이고, 2분 이내에 AI 활동이 명확하게 보이는 데모

**보조 사용자: 개발자 본인 (최승훈)**
- 시스템 구축에서 실질적인 학습 가치 필요
- 전문 프로젝트에 이전 가능한 코드 패턴 습득 원함 (멀티 에이전트 오케스트레이션, 3D 렌더링 최적화, 실시간 상태 관리)

## 5. 제품 범위

### MVP 기능 (v1.0)

#### F-01: 3D 공장 바닥 시각화
- 20×20 격자 공장 바닥에 5개의 고정 기계 렌더링 (프레스, CNC, 컨베이어 유형)
- 3개의 AMR 로봇이 바닥에서 부드럽게 이동
- 기계 메시는 상태에 따라 색상 변경: 파랑(정상), 주황(저하), 빨강(고장)
- 로봇 메시는 목표 위치로 부드러운 보간 이동
- 씬은 24시간 연속 운영에서 안정적이고 메모리 누수 없어야 함

#### F-02: 실시간 센서 스트림
- 백엔드와 WebSocket 연결, ~10Hz로 스트리밍
- 기계당 1개씩 총 5개의 ECharts 라인 차트, 각각 시간에 따른 진동(Hz) 표시
- 프레임 예산 충돌 방지를 위해 4Hz로 스로틀된 차트 업데이트
- 기록 링 버퍼: 기계당 최근 300개 데이터 포인트 (30초 분량)

#### F-03: 고장 주입 및 이상 감지
- 백엔드가 60~120초마다 임의의 기계에 자동으로 고장 주입
- 고장 상태는 30초 지속 후 해제
- 고장 감지 시 UI에 즉시 빨간색 AlertBanner 표시

#### F-04: 멀티 에이전트 AI 체인
- **에이전트 A (진단):** Claude API를 통해 30초 센서 윈도우를 분석, 고장 유형·심각도·신뢰도 분류
- **에이전트 B (라우팅):** Claude API를 통해 가장 가까운 유휴 로봇을 선택하고, 고장 기계까지의 경유지 경로 계산
- **에이전트 C (의사결정):** Claude API를 통해 세 가지 조치 옵션(즉시 수리, 예약 정비, 임시 우회)에 대한 RICE 점수 계산

#### F-05: 에이전트 체인 UI 패널
- 실시간 진행 표시: 각 에이전트가 "running"(펄스 애니메이션) → "complete"(요약 텍스트) 순으로 표시
- 파견 상태 카드: 어떤 로봇이 어떤 기계로 향하는지 ETA와 함께 표시
- 에이전트 패널은 지속적으로 표시; 최근 9개 에이전트 이벤트 보여줌 (체인 3회 실행 분량)

#### F-06: 로봇 파견 애니메이션
- 에이전트 B가 파견 명령을 실행하면, 대상 로봇의 위치가 Three.js에서 경유지 경로를 따라 보간
- 이동 중 로봇 상태가 "dispatched"로 변경 (색상 변화)

### Post-MVP 기능 (v2.0 백로그)
- A* 경로 탐색 및 로봇 간 충돌 회피
- 기계 유형별 3D 모델 (균일한 박스 대신)
- 에이전트 보고서 이력: 모든 과거 사건의 스크롤 가능한 로그
- 50대 이상 기계 확장을 위한 WebGL 인스턴스 렌더링
- Three.js 씬의 모바일 반응형 레이아웃

## 6. 기술 제약사항

| 제약 | 요구사항 |
|---|---|
| Claude API | `claude-sonnet-4-6` 모델 사용; 각 에이전트 호출은 10초 이내에 완료 |
| WebSocket | 클라이언트당 단일 영구 연결; 백엔드는 재연결을 우아하게 처리 |
| 메모리 | 1시간 운영 후 Three.js 씬의 힙 메모리가 200MB 초과 금지 |
| 배포 | 백엔드는 Railway 무료 티어($5/월); 프론트엔드는 Vercel 무료 티어 |
| 브라우저 지원 | 최신 Chrome/Edge, 최신 Firefox; IE 또는 Safari <16 지원 불필요 |

## 8. 인수 기준

### AC-01: 3D 씬
- [ ] 일반 노트북에서 3초 이내에 페이지 로딩 및 Three.js 씬 렌더링
- [ ] 5개 기계 메시와 3개 로봇 메시 가시화
- [ ] 60분 연속 운영 후 브라우저 힙 메모리가 초기 대비 20% 이내

### AC-02: 센서 데이터 파이프라인
- [ ] ECharts 차트가 250ms마다 눈에 띄게 업데이트
- [ ] 차트 이력이 최소 30초 분량의 데이터 표시
- [ ] 차트 업데이트 중 Three.js에서 프레임 드롭 없음

### AC-03: 에이전트 체인
- [ ] 에이전트 A가 고장 주입 후 5초 이내에 실행
- [ ] 에이전트 B가 에이전트 A 완료 후 실행
- [ ] 에이전트 C가 에이전트 B 완료 후 실행
- [ ] 세 에이전트 모두 고장 주입 후 40초 이내에 Agent Panel에 "complete" 상태 표시
- [ ] Claude API 호출 타임아웃 시 크래시 없이 폴백 메시지 표시

### AC-04: 배포
- [ ] 공개 URL 접근 가능
- [ ] 프로덕션에서 WebSocket 연결 성공 (wss://)
- [ ] `/health` 엔드포인트가 500ms 이내 응답
- [ ] UptimeRobot이 백엔드를 웜 상태로 유지; 방문자에게 콜드 스타트 지연 없음

## 10. 리스크 및 대응

| 리스크 | 발생 가능성 | 영향도 | 대응 방안 |
|---|---|---|---|
| Claude API 응답 파싱 실패 | 중간 | 낮음 | 세 에이전트 모두 정규식 기반 JSON 추출 + 구조화된 폴백 |
| Three.js 메모리 누수 (프로덕션) | 중간 | 높음 | 언마운트 시 `disposeScene()` 호출 + 지오메트리/머티리얼 캐시로 중복 방지 |
| Railway 무료 티어 슬립 | 높음 | 높음 | UptimeRobot이 5분마다 `/health` 핑 |
| Claude API 레이턴시 > 10초 | 낮음 | 중간 | 에이전트 호출당 `asyncio.timeout(10)` 적용; 폴백이 정상적으로 반환 |
| WebSocket 폭주로 인한 UI 끊김 | 중간 | 중간 | RAF 드레인 큐가 WS I/O를 React 렌더 사이클에서 분리 |
