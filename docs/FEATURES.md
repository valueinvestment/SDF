# SDF 플랫폼 실제 사용 가능 기능 명세서 (Product Feature Specifications)

SDF 디지털 트윈 빌더 플랫폼에서 사용자가 실시간으로 조작하고 검증할 수 있는 핵심 런타임 기능 명세입니다. 본 플랫폼은 클라이언트 단의 고성능 60fps Three.js 렌더 루프 및 확장형 전역 상태 엔진을 기반으로 작동합니다.

---

## ⚙️ 1. 3D 노코드 저작 도구 & 공간 배치 (3D Interactive Space Creator)

사용자는 화면 내에서 고정된 3D 씬을 보는 것을 넘어, 자유롭게 공간을 편집하고 레이아웃을 마스터할 수 있습니다.

* **기기 추가 및 정밀 스케일 편집**: 상단 및 사이드바 팰릿을 통해 새로운 장비(Press, CNC, Conveyor, Robot)를 실시간 추가할 수 있습니다. 기기 추가 시 가로, 세로, 높이 배율(Scale Factor)을 슬라이더로 조절하여 커스텀 크기로 인스턴스화할 수 있습니다.
* **외부 3D 모델 동적 인젝션 (GLB/GLTF Upload)**: 표준 3D 파일(`.glb` / `.gltf`)을 드래그 앤 드롭으로 업로드하거나 외부 URL을 입력하여 씬에 직접 인스턴스화할 수 있습니다. 업로드된 모델은 `GLTFLoader`를 거쳐 바운딩 박스 기준 자동 스케일·바닥 정렬되며, 기존 빌트인 장비와 동일하게 `TransformControls` 기즈모와 격자 스냅에 바인딩되어 위치/회전/크기를 편집할 수 있습니다. (파일은 브라우저 메모리 ObjectURL로 처리되어 서버 저장 불필요)
* **마우스 드래그 기즈모 (TransformControls)**: 저작 모드 활성화 후 3D 공간 상의 설비 메쉬를 클릭하면 이동 화살표 기즈모가 부착되어 마우스로 직관적인 위치 수정이 가능합니다.
* **자석형 격자 스냅 (Grid Snapping)**: 3D 설비 드래그 이동 시 X, Z축 기준 0.5 또는 1.0 격자 단위로 딱딱 맞물려 정렬되는 자석 스냅 수학 로직이 작동하여 정밀한 대열 배치가 가능합니다. 수정 완료 즉시 백엔드 시뮬레이터와 `sync_entities` 웹소켓 통신을 통해 좌표계가 동기화됩니다.
* **카메라 뷰포트 영속성 (Viewport Memory)**: 사용자가 마우스 드래그 및 줌을 통해 설정한 Three.js OrbitControls의 최적의 카메라 좌표(`position`)와 초점 지점(`target`) 벡터 데이터를 전역 상태 엔진이 실시간 캡처하여 저장합니다.

## 📊 2. GUI 자유 레이아웃 커스터마이저 (Dynamic Layout Manager)

고정된 그리드 컴포넌트 환경을 탈피하여 전체 화면 배치를 런타임에 재구성합니다.

* **드래그 앤 드롭 가변 그리드 (react-grid-layout)**: 메인 화면의 3D 공장 floor 영역, 하단 실시간 차트 영역, 우측 AI 에이전트 패널, 상세 제어 컴포넌트를 마우스 드래그로 자유롭게 이동하고 우하단 핸들을 잡아 크기를 조절할 수 있습니다. 컬럼 수(2/3/4열) 전환과 패널 표시/숨김도 지원합니다. 좌표 배치 상태(`x/y/w/h`)는 `dashboardConfig` 전역 스토어에 실시간 기록되어 URL/로컬 저장소로 영속화됩니다. (deprecated `WidthProvider` 대신 ResizeObserver 기반 `useContainerWidth` 훅 사용)
* **차트 프리셋 커스텀**: 각 설비별 실시간 차트 위젯에서 차트의 형태(Line, Bar, Radar 차트 등), 모니터링할 센서 데이터 종류(진동, 온도, 전류 등), 그리고 라인 및 배경의 네온 컬러 헥사코드를 UI에서 실시간 매핑 및 변경할 수 있습니다.
* **모델별 상세 속성 정의**: 각 컴포넌트 상세보기 창에서 기계 모델별 커스텀 네이밍, 담당 작업자 정보, 관리 부서 등 고유 메타 속성 데이터를 텍스트 인풋을 통해 직접 주입하고 라벨링할 수 있습니다.

## 📈 3. 변환 레이어 및 가상 시뮬레이터 (Data Pipeline & Simulator)

실시간 데이터 스트림을 입맛대로 가공하고, 오프라인 환경에서도 완벽한 시뮬레이션을 구동합니다.

* **런타임 데이터 트랜스포메이션**: 백엔드 웹소켓에서 수신한 Raw 센서 데이터가 차트에 그려지기 전, 사용자가 입력한 문자열 수식(예: `(vibration * 1.2) + (temperature / 2)`)을 안전한 파싱 토큰 연산을 통해 가공하여 새로운 '커스텀 계산 지표(Computed Metric)'를 실시간 차트 선형 데이터로 바인딩합니다.
* **가우시안 노이즈 모킹 엔진**: 외부 웹소켓 서버와 연결되지 않은 독립 데모 모드일 경우, 사용자가 입력한 최소(min), 최대(max), 평균(avg) 제어 변수를 바탕으로 프론트엔드 내부 타이머가 실시간 사인파 데이터를 자동 생성합니다. 여기에 실제 물리 환경과 유사한 가우시안 백색 소음(White Noise) 필터 분산 값을 동적으로 섞어줄 수 있습니다.
* **시뮬레이션 배속 컨트롤 박스**: 데이터가 쌓이는 주기 및 시뮬레이션 고장 발생 간격을 속도 제어기(1x, 2x, 5x 배속)를 통해 가속할 수 있어, 짧은 시간 내에 대규모 트렌드 누적 및 규칙 엔진 테스트가 가능합니다.

## 🏭 4. ISA-95 표준 가상 MES 및 동적 룰 엔진 (Enterprise BI & Rules)

산업 생산 시스템 환경과 결합하여 비즈니스 가치가 결합된 AI/BI 시나리오를 구동합니다.

* **가상 작업 지시서 (WorkOrder) 연계**: 각 장비 인스턴스에 ISA-95 표준 규격을 벤치마킹한 데이터 구조(생산 지시 번호, 품명, 작업 우선순위[S/A/B], 목표 수량, 현재 양품 생산 수량, 납기 기한)를 바인딩하여 대시보드 상단 현황판 위젯에 실시간 갱신합니다. 설비가 정상 가동되면 가상 생산 수량이 실시간 누적 증가합니다.
* **조건부 룰 엔진 바인딩**: 데이터 가공 수식 또는 실시간 센서 값이 특정 임계치를 돌파할 때(예: `temperature > 100`) 발동할 규칙을 지정합니다. 트리거 시 해당 3D 설비 메쉬에 붉은색 네온 오버레이를 즉시 입히고 알림 배너를 노출합니다.
* **헤드리스 액션 러너**: 룰 트리거와 연동되어 UI 외부 동작을 수행합니다. (1) Web Audio API 합성음을 활용해 경고 신호음을 브라우저 상에 사운드로 재생하며, (2) 사용자가 입력한 외부 Slack 또는 Discord Webhook URL 경로로 장애 스냅샷 리포트 JSON 패킷을 실시간 `fetch()` POST 발송합니다.
* **Closed-Loop 공정 우회 배정 (Rerouting)**: 특정 설비에 심각한 결함(`high fault`)이 발생하거나 사용자가 설비를 긴급 정지시키면, 해당 장비에 할당되어 있던 잔여 중요 `WorkOrder` 스케줄이 실시간으로 공장 내 인근 유휴(idle) 상태인 동일 사양 장비의 작업 큐(Queue)로 자동 이관됩니다. 이 흐름은 메인 대시보드의 [MES 실시간 작업 모니터링 현황판]에 애니메이션으로 가시화됩니다.

## 🛡️ 5. 결함 격리 및 인코딩 공유 (Resilience & Shareability)

플랫폼의 안정성을 보장하고 커스텀 설정 상태를 링크 하나로 완벽하게 전파합니다.

* **대시보드 위젯 에러 바운더리 (Fault Isolation)**: 사용자가 데이터 트랜스포메이션 수식을 잘못 작성하거나(문법 오류), 서드파티 플러그인 컴포넌트 렌더링 중 런타임 크래시 에러가 발생하더라도, 메인 3D factory 캔버스나 플랫폼 전체 시스템은 중단 없이 정상 구동됩니다. 에러가 발생한 해당 차트/패널 영역만 React `ErrorBoundary`로 안전하게 격리되어 '설정을 확인해주세요'라는 친절한 네온풍 가이드 뷰포트로 치환됩니다.
* **JSON 파일 기반 Export/Import**: 마우스 클릭 한 번으로 사용자가 커스텀 배치한 3D 공간 좌표, UI 레이아웃 크기, 색상, 커스텀 수식, 가상 MES 데이터 일체를 단일 `.json` 파일로 로컬 PC에 저장(Export)하고, 언제든 다시 업로드(Import)하여 대시보드 상태를 100% 복구할 수 있습니다.
* **공유 가능한 URL 상태 인코딩 (Shareable URL)**: 설정 JSON 오브젝트 전체를 `lz-string` 알고리즘을 사용해 압축한 뒤 브라우저 URL의 쿼리 스트링(`?config=...`)에 실시간 동기화합니다. 사용자는 이 인코딩된 URL 링크를 복사하여 타인에게 전달하는 것만으로, 내가 설계한 3D 구도와 대시보드 설정을 상대방 브라우저 화면에 그대로 복제하여 띄워줄 수 있습니다.
* **URL 길이 방어 및 localStorage 자동 우회 (Defensive Serialization)**: 커스텀 3D 모델·자유 레이아웃으로 설정 JSON이 비대해져 `lz-string` 압축 결과가 안전 기준선(4,000자)을 초과하면, URL 쿼리 동기화를 차단하고 브라우저 `localStorage`로 자동 저장(Fallback)하면서 사용자에게 Toast 경고를 노출합니다. 페이지 로드 시 URL 파라미터 → localStorage 순으로 설정을 복원하여 거대한 설정도 안전하게 유지됩니다. (검증 로직은 순수 함수 `lib/configSerialization.ts`로 분리되어 단위 테스트됨)
